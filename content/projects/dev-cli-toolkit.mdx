## Why

Every new side project started the same way: copy `.env`, set up GitHub,
configure deploys, write the same `README` skeleton. I wanted one command.

## What it does

- `kit new <name>` &mdash; clones a chosen template, swaps placeholders, and
  initializes a fresh git repo.
- `kit env` &mdash; interactive prompt that fills in `.env` from a typed
  schema and validates required keys.
- `kit deploy` &mdash; thin wrapper over the Vercel CLI plus a sanity check
  that the project actually builds locally first.

```bash
kit new portfolio --template next-mdx
cd portfolio
kit env
kit deploy
```

## What I learned

Designing CLIs that coding agents (and tired humans at 1am) can use reliably
is its own discipline: every command needs `--help` with examples, predictable
exit codes, and machine-parseable output as an opt-in.
